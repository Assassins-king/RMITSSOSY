消息处理

在开始学习内容提供者之前，我们先来学习一下消息处理。

在进一步学习之前，请浏览下面的问题：

  1. 什么是UI线程？为什么不要在UI线程中开比较费时的工作？
  2. 什么是MessageQueue？消息队列由什么进行管理，如何获取消息队列？
  3. 对于Looper的创建，主线程和非主线程什么区别，Looper同消息队列什么关系？如何获取Looper对象？
  4. 什么是Message？如何获取消息对象？为什么不用new来创建消息对象？
  5. 什么是Handler？如何将消息传递给Looper？然后Looper怎么做？
  6. Looper、MessageQueue、Handler这三者是怎么协同工作的？
  7. obtainMessage方法中的四个参数的意义？

在android系统中，每个Android应用程序都运行在一个dalvik虚拟机进程中，进程开始的时候会启动一个主线程(MainThread)，主线程负责处理和ui相关的事件，因此主线程通常又叫UI线程。而由于Android采用UI单线程模型，所以只能在主线程中对UI元素进行操作。如果在非UI线程直接对UI进行了操作，则会报错：

CalledFromWrongThreadException:only the original thread that created a view hierarchy can touch its views

Android为我们提供了消息循环的机制，我们可以利用这个机制来实现线程间的通信。那么，我们就可以从非UI线程发送消息到UI线程，最终让UI线程来进行UI的操作。

还有就是，对于运算量较大的操作，如： IO操作，我们需要新开线程来处理这些繁重的工作，以免阻塞UI线程。

1. Android的消息处理机制

Android的线程消息处理有几个相关类：Handler、Message、Looper、MessageQueue等。

  1. MessageQueue： 是一种 数据 结构，是FIFO的消息队列，存放Message，但并非实际意义的保存，而是将Message以链表的方式串联起来的，由Looper来进行调度。每一个线程最多只可以拥有一个MessageQueue数据结构。
    创建一个线程的时候，并不会自动创建其MessageQueue。通常使用一个Looper对象对该线程的MessageQueue进行管理。主线程创建时，会创建一个默认的Looper对象，而Looper对象的创建，将自动创建一个Message Queue。其他非主线程，不会自动创建Looper，在需要的时候，通过调用Looper.prepare方法来实现。
    Message不是直接加入消息队列的，而是由同Looper相关联的Handler来发送的。
    可以使用Looper.myQueue()来获取当前线程的MessageQueue
  2. Message： 消息对象，Message Queue中的存放的对象。一个Message Queue中包含多个Message。
    Message类中包含一个对消息的描述和要传递给Handler的任意数据。该类的对象中包含两个int域和一个对象域
    虽然Message的构造方法是public的，但是要获取Message对象的最好方式是调用Message.obtain()或者是Handler.obtainMessage()，这样会从回收池中重利用回收对象。
    调用handler的removeMessages()时，将Message从Message Queue中删除，同时放入到回收池中。
  3. Looper： 是MessageQueue的管理者，每一个MessageQueue都不能脱离Looper而存在。创建一个Looper对象时，会同时创建一个MessageQueue对象。同时每一个Looper对象和一个线程关联。Looper的字面意思是“循环者”，它被设计用来使一个普通线程变成Looper线程。所谓Looper线程就是循环工作的线程。在程序开发中（尤其是GUI开发中），我们经常会需要一个线程不断循环，一旦有新任务则执行，执行完继续等待下一个任务，这就是Looper线程。通过调用Looper.myLooper()可以获得当前线程的Looper对象
    除了主线程有默认的Looper进行消息处理以外，其他线程默认是没有消息处理能力的，所以，不能接受Message。若需要处理消息，在线程中通过调用Looper.prepare()来创建消息队列，同时会有一个管理它的Looper，这样该线程就有了自己的Looper对象和MessageQueue数据结构了。然后，在调用Looper.loop()来循环处理消息。
    Looper只是MessageQueue的管理者，而同消息队列进行交互的角色由Handler扮演。Looper从MessageQueue中取出Message，然后，交由Handler的handleMessage进行处理。处理完成后，调用Message.recycle()将其放入回收池中。
  4. Handler： 消息的处理者，handler 负责将需要传递的信息封装成Message，通过调用handler 对象的obtainMessage()来实现； 将消息传递给Looper，这是通过handler 对象的sendMessage()来实现的。继而由Looper将Message放入MessageQueue中。 当Looper对象看到MessageQueue中含有Message，就将其广播出去。该handler 对象收到该消息后，调用相应的handler 对象的handleMessage()方法对其进行处理。

线程通过Looper建立自己的消息循环， Looper负责从MessageQueue中取出消息，并且分发到消息指定目标Handler对象。Handler对象绑定到线程的局部变量Looper，负责Message的发送及处理。使用Handler时，需要实现handleMessage(Message msg)方法来对特定的Message进行处理，例如更新UI等。

2. 示例

我们来看一个Android线程通讯的一个例子，这个例子实现点击按钮之后从主线程发送消息"hello"到另外一个线程。
主程序如下：

public class MainActivity extends AppCompatActivity {
	//定义消息码
    private final int MSG_HELLO = 0;
    private Handler handler;
    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        //启动一个looper线程
        new Thread(){
            public void run(){
				//初始化looper
                Looper.prepare();
				//绑定handler到本线程的looper，在Handler的构造方法中
                //会获取本线程的Looper对象，并存放在自己的成员变量中
                handler = new Handler(){
					//定义处理消息的方法
                    public void handleMessage(Message msg){
						//区别处理消息的类别
                        switch (msg.what){
                            case MSG_HELLO: Log.d("Test","thread receive msg:"+msg.obj);
                        }
                    }
                };
				//启动消息循环
                Looper.loop();
            }
        }.start();
    }

    public void click(View view){
        String str = getString(R.string.app_name);
        Log.d("Test","is ready to send msg:"+str);
        handler.obtainMessage(MSG_HELLO,str).sendToTarget();
    }
}

上例中的代码还可以改变为，发送方向从子线程向主线程发送：

public class MainActivity extends AppCompatActivity {

    private final int MSG_HELLO = 0;
    private Handler handler;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
		//此时的handler不是同子线程绑在一起，而是同主线程绑在一起
        handler = new Handler() {
            public void handleMessage(Message msg) {
                switch (msg.what) {
                    case MSG_HELLO:
                        Log.d("Test", "thread receive msg:" + msg.obj);
                }
            }
        };

        new Thread(){
            public void run(){
                handler.sendMessage(handler.obtainMessage(MSG_HELLO,getString(R.string.app_name)));
            }
        }.start();
    }
}

3. 主要类的介绍

结合上面的示例，我们介绍一下主要使用的类

3.1. Message类

3.1.1. 常用域

public int arg1,arg2

这两个域是替代setData()方法的低成本方式，可存放两个整型数据。

public Object obj

存放发送给接收者的任意对象，其他种类数据的传送，使用setData(Bundle).

public int what

用户定义的消息代码，通过该消息码，接收者可以识别该消息是什么消息。每个Handler都有自己的消息代码的命名空间，所以你不必担心你的消息码同其他的handler相冲突。

3.2. Looper

典型的程序示例：

class LooperThread extends Thread {
    public Handler mHandler;

    public void run() {
      Looper.prepare();

      mHandler = new Handler() {
          public void handleMessage(Message msg) {
              // process incoming messages here
          }
      };

      Looper.loop();
    }
}

3.3. Handler

Handler使得你可以发送和处理线程的消息队列中的消息。每个handler对象同某个线程及它的消息队列相关。当你创建一个新的handler的时候，该handler同创建它的线程关联在一起，从此，就可以向消息队列发送消息或者Runable对象，还可以从消息队列中出队消息。

Handler的两个主要用途在于: (1) 安排消息并且在未来的某个时候执行某个Runable对象；(2)排队在其他线程要执行的动作

安排消息是通过执行post(Runnable), postAtTime(Runnable, long), postDelayed(Runnable, long), sendEmptyMessage(int), sendMessage(Message), sendMessageAtTime(Message, long), 和 sendMessageDelayed(Message, long) 方法来完成的。

post方法用来排队消息队列中的Runable对象，send方法用来排队消息。当post或者send到一个handler以后，既可以立刻处理该项，也可以指定一定的延迟

当你的应用的进程创建以后，它的主线程专门运行一个消息队列来管理顶层的应用程序对象（activities、broadcast receivers等），你可以创建自己的线程同主应用的线程通过handler相交互。这也是通过post或者send完成的。
